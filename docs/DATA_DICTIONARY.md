# Data Dictionary

## Dataset Overview
This workspace contains customer and transaction sample data used for ingestion, profiling, and workflow exercises.
Last Updated: 2025-05-21
Maintained By: Data Engineering Team

## Column to KPI Mapping

### Customer Count
- **Formula**: COUNT(customer_id)
- **Related Columns**: customer_id, signup_date
- **Why It Matters**: Tracks how many customer records are available for analysis and acquisition reporting
- **Update Frequency**: Daily

### Monthly Revenue
- **Formula**: SUM(amount)
- **Related Columns**: amount, transaction_date, id
- **Why It Matters**: Measures total revenue captured from transactions in a reporting period
- **Update Frequency**: Daily

### Sales Velocity
- **Formula**: COUNT(id) / days
- **Related Columns**: id, transaction_date
- **Why It Matters**: Measures the rate of transaction activity over time
- **Update Frequency**: Weekly

### Contactability Rate
- **Formula**: COUNT(email present) / COUNT(customer_id)
- **Related Columns**: email, customer_id
- **Why It Matters**: Shows how reachable the customer base is for marketing and service outreach
- **Update Frequency**: Weekly

### Transaction Completion Rate
- **Formula**: COUNT(status = completed) / COUNT(id)
- **Related Columns**: status, id, transaction_date
- **Why It Matters**: Indicates how much of the recorded activity is successfully completed
- **Update Frequency**: Daily

### Demographic Coverage
- **Formula**: COUNT(age present) / COUNT(customer_id)
- **Related Columns**: age, customer_id
- **Why It Matters**: Helps the team understand how complete the customer profile is for segmentation
- **Update Frequency**: Monthly

## Ambiguous Columns & Resolutions

### Column: name
- **Original Ambiguity**: Does this refer to the customer, the company, or the contact person?
- **Resolved Meaning**: Customer full name used for account lookup and service workflows
- **Business Interpretation**: Human-readable label for the customer record
- **Proposed Rename**: customer_name
- **Risk If Misunderstood**: Analysts may join or group records on the wrong entity and create duplicate customer counts

### Column: amount
- **Original Ambiguity**: Is this transaction value, account balance, or a fee amount?
- **Resolved Meaning**: Gross transaction amount in USD captured at the time of the record
- **Business Interpretation**: Revenue generated by a single transaction
- **Proposed Rename**: transaction_amount_usd
- **Risk If Misunderstood**: Revenue reporting and pricing analysis can be overstated or understated

### Column: status
- **Original Ambiguity**: Does this represent customer status, transaction status, or pipeline status?
- **Resolved Meaning**: Operational state of the record; must be validated against the source system before analysis
- **Business Interpretation**: Used to separate active work from incomplete or pending records
- **Proposed Rename**: record_status
- **Risk If Misunderstood**: Completion-rate analysis and workflow reporting become unreliable

## Column Relationships

### Revenue per Customer
- **Definition**: SUM(amount) grouped by customer_id
- **How It Matters**: Identifies high-value customers for retention and upsell prioritization
- **Example**: The top 10 percent of customers can contribute a disproportionate share of revenue
- **Related Columns**: customer_id, amount, transaction_date

### Revenue Over Time
- **Definition**: Rolling sum of amount over transaction_date windows
- **How It Matters**: Reveals momentum, seasonality, and changes in sales performance
- **Example**: Weekly revenue trending up after a campaign launch
- **Related Columns**: amount, transaction_date, id

### Customer Profile Completeness
- **Definition**: Percentage of customer records with email and age populated
- **How It Matters**: Determines how usable the customer base is for segmentation and outreach
- **Example**: Higher completeness supports more accurate segmentation and contact campaigns
- **Related Columns**: customer_id, email, age

## Column Details

### customer_id
- **Type**: Integer
- **Business Meaning**: Unique customer identifier from the CRM system
- **Example**: 1
- **Null Handling**: Never null; primary key
- **Related KPI**: Customer count, lifetime value calculation
- **Updates**: Assigned when the customer is created

### name
- **Type**: String
- **Business Meaning**: Customer full name
- **Example**: Alice
- **Null Handling**: Should be populated for active customers
- **Related KPI**: Customer lookup and service resolution
- **Updates**: Updated if the customer changes their legal or preferred name

### email
- **Type**: String
- **Business Meaning**: Primary customer contact email
- **Example**: alice@example.com
- **Null Handling**: May be missing in raw data and should be validated before outreach
- **Related KPI**: Contactability rate, marketing reach
- **Updates**: Updated when the customer changes their contact details

### signup_date
- **Type**: Datetime
- **Business Meaning**: Date the customer entered the system
- **Example**: 2025-01-15
- **Null Handling**: Should not be null for newly onboarded customers
- **Related KPI**: Acquisition cohorts, customer growth
- **Updates**: Set once at onboarding

### id
- **Type**: Integer
- **Business Meaning**: Unique transaction or event identifier
- **Example**: 1
- **Null Handling**: Never null in a clean transaction feed
- **Related KPI**: Transaction count, audit trail completeness
- **Updates**: Assigned when the transaction is recorded

### amount
- **Type**: Float
- **Business Meaning**: Revenue recorded for a transaction
- **Example**: 150.99
- **Null Handling**: Rare; investigate missing or negative values immediately
- **Related KPI**: Monthly revenue, average transaction value, revenue per customer
- **Updates**: Set when the transaction is completed

### status
- **Type**: String
- **Business Meaning**: Lifecycle state of the record
- **Example**: completed
- **Null Handling**: Should be reviewed if missing or numeric-looking values appear
- **Related KPI**: Completion rate, operational throughput
- **Updates**: Determined by the source workflow

### age
- **Type**: Integer
- **Business Meaning**: Customer age in years
- **Example**: 34
- **Null Handling**: Optional demographic field
- **Related KPI**: Demographic coverage, segmentation
- **Updates**: Updated only when customer profile changes

### transaction_date
- **Type**: Datetime
- **Business Meaning**: Date the transaction occurred
- **Example**: 2025-01-01
- **Null Handling**: Should be present for all valid transactions
- **Related KPI**: Sales velocity, revenue over time
- **Updates**: Captured at transaction time
